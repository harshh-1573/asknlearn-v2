🧠 What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around objects — instances of classes that bundle data (attributes) and behavior (methods) together.
The goal is to make code modular, reusable, and easier to maintain.

⚙️ Core OOP Concepts
1. Class

A class is a blueprint or template for creating objects.

It defines attributes (variables) and methods (functions) common to all objects of that type.

Example idea: A class “Car” defines properties like color, model, speed, and behaviors like start() or brake().

2. Object

An object is an instance of a class.

It represents a real-world entity with a unique identity, state, and behavior.

Each object can have different data while sharing the same structure defined by the class.

3. Encapsulation

Encapsulation means binding data and methods together into a single unit (the class).

It also involves restricting direct access to an object’s data to protect it from unintended modification.

Achieved using access modifiers (like private, public, protected).

Helps in data hiding and security.

4. Abstraction

Abstraction means showing only the essential features of an object and hiding unnecessary details.

It focuses on what an object does, not how it does it.

Example: You use a car’s steering wheel without needing to know the mechanism inside.

5. Inheritance

Inheritance allows a class to acquire properties and behaviors of another class.

The new class is called the subclass (child), and the existing class is the superclass (parent).

Promotes code reusability and hierarchical relationships.

Types of Inheritance:

Single Inheritance – one class inherits from one parent.

Multilevel Inheritance – class inherits from a child class of another.

Hierarchical Inheritance – multiple classes inherit from one parent.

Multiple Inheritance – class inherits from multiple parents (supported in some languages).

Hybrid Inheritance – combination of different inheritance types.

6. Polymorphism

Polymorphism means “many forms.”

It allows one interface to be used for different types of actions.

In simple terms, the same method can behave differently based on the object that calls it.

Types of Polymorphism:

Compile-time (Static) Polymorphism – method overloading or operator overloading.

Runtime (Dynamic) Polymorphism – method overriding using inheritance.

7. Constructor and Destructor

Constructor: A special method used to initialize objects when they are created.

Destructor: Used to clean up resources when an object is destroyed.

These support object lifecycle management.

8. Association, Aggregation, and Composition

These define relationships between classes:

Association – a general link between two independent classes (e.g., teacher and student).

Aggregation – a “has-a” relationship, where one class contains another but both can exist independently (e.g., department has professors).

Composition – a strong “has-a” relationship where one class cannot exist without the other (e.g., house has rooms).

9. Message Passing

Objects communicate with each other using messages (usually method calls).

This allows interaction between different parts of a program.

10. Dynamic Binding

The process of linking a method call to its definition at runtime (rather than compile-time).

Enables runtime polymorphism.

11. Modularity

Dividing a program into independent modules (classes).

Each module performs a specific function, making it easier to maintain, debug, and reuse.

✅ Advantages of OOP

Code reusability (through inheritance)

Data security (through encapsulation)

Flexibility and scalability

Easier maintenance and debugging

Real-world modeling

Extensibility through polymorphism
